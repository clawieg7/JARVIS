# Regels: Inbox Verwerking

Wanneer Gijs iets deelt (link, idee, vraag, screenshot, Instagram post, etc.) — verwerk het als volgt:

## Stap 1: Analyseer
- Wat is het? (artikel, tool, idee, taak, inspiratie)
- Zoek uit wat het inhoudt — gebruik internet als nodig
- Train jezelf: word beter in dit type input herkennen

## Stap 2: Key points
- Schrijf kort de kern op
- Wees langer en vollediger als de input dat vereist
- Leer van wat Gijs vervolgens doet: was je analyse goed? Pas aan.

## Stap 3: Context mapping
- Hoort dit bij een bestaand project? Welk?
- Hoe voegt dit toe aan bestaande projecten?
- Past dit in huidige to-do, agenda, projecten?
- Hoe helpt dit lange termijn doelen bereiken?
- Prioriteer: urgent / belangrijk / nice-to-have
- Tijdsinschatting: hoeveel tijd kost dit?
- Timing: wanneer is het beste moment?

## Stap 4: Voorstellen
- Minimaal 3 concrete vervolgstappen (praktijkgericht)
- Denk mee: wat mis ik? Wat kan beter?

## Stap 5: Vooruitwerken
- Werk alvast uit wat je kunt op template/branch basis
- Maak het direct integreerbaar — zodat Gijs alleen akkoord hoeft te geven
- Git branch principe: klaar om te mergen, niet nog uitwerken na akkoord
- Bij geen akkoord: geen werk verloren, branch blijft staan

## Toepassing
- Dit geldt voor ELKE inbox.md in elk project
- Dit geldt voor berichten via Telegram
- Dit geldt voor links, screenshots, ideeën die Gijs deelt
- Leer en verbeter continu: analyseer of je voorbereiding goed was
